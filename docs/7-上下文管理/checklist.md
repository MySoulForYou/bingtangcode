# 上下文管理（Context Management）验收清单

## 1. 轻量预防验收

- [ ] **单个工具结果超限落盘**
  - **测试输入**：构造一个工具执行，返回 60,000 字节（大于 50,000 字节阈值）的文本数据。
  - **预期观测**：
    - 在项目目录的 `.bingtangcode/sessions/<sessionId>/tool-results/` 下生成了对应的工具落盘文件。
    - 检查发送给 API 的消息，该 `ToolResult` 内容被替换，且包含前 20 行或前 2048 字节头部预览、落盘的绝对路径、原始大小，且没有尾部预览。
    - 替换后的内容中必须包含指示模型重新读取的重读提示。

- [ ] **单条消息合计超限落盘**
  - **测试输入**：单条消息返回三个工具结果，大小分别为 `120,000 字节`、`90,000 字节`、`30,000 字节`（合计 240,000 字节，均小于 50,000 单个阈值，但合计超过 200,000 字节）。
  - **预期观测**：
    - 按字节数从大到小排序，排在第一的 `120,000 字节` 工具结果触发落盘。
    - 落盘后，剩余未落盘结果大小为 `90,000 + 30,000 = 120,000 字节`，小于等于 200,000 字节，停止落盘。
    - 最终该条消息的 `ToolResult` 中，只有 `120,000 字节` 的那个结果内容被替换成了落盘预览格式，其余两个结果保持原样。

## 2. 重量兜底与摘要生成验收

- [ ] **LLM 摘要草稿剥离**
  - **测试输入**：模拟 LLM 返回带有草稿的摘要响应，格式为 `<draft>思考与草稿内容</draft>\n\n## 1 主要请求和意图\n...`。
  - **预期观测**：
    - 程序处理后的摘要消息中不包含 `<draft>` 和 `</draft>` 标签及其包含的任何草稿内容。
    - 摘要内容首行不含多余的前导空白。

- [ ] **摘要结构完整性与原文保留**
  - **测试输入**：对话历史包含多轮对话（包含用户提问、模型回答和工具返回），触发摘要。
  - **预期观测**：
    - 最终生成的摘要中，九个二级标题 `## 1 主要请求和意图` 到 `## 9 可能的下一步` 必须全数存在。
    - 检查 `## 6 所有用户消息原文` 下，所有被移去的历史 `USER` 消息原文被原样且完整地列在此板块下。

- [ ] **近期历史保留**
  - **测试输入**：设定保留下限为 token 数累积 10,000，且保留消息数不少于 5 条。
  - **预期观测**：
    - 历史消息中，最新的几条消息（满足累积 token 估算数 >= 10000 且条数 >= 5）将以原文形式保留，不被压缩。

- [ ] **边界提示消息合并**
  - **测试输入**：重量压缩执行完毕。
  - **预期观测**：
    - 检查替换后的对话历史，紧随在摘要内容之后被追加了如下格式的边界提示，且整体作为一个 `USER` 消息保存：
      ```text
      📖 重要提示：
      - 需要文件原文时，请使用 ReadFile 工具重新读取对应路径
      - 需要错误原文或用户原话时，请追溯摘要中的记录或重新读取
      - 不要依据摘要内容做猜测性补全，摘要仅作为索引而非精确原文
      ```

## 3. 触发与控制验收

- [ ] **安全余量自动触发**
  - **测试输入**：在配置文件中设定 `context_window: 200000`。构造多轮对话使上下文估算 token 达到 `167,500`（大于安全阈值 `200000 - 20000 - 13000 = 167000`）。
  - **预期观测**：
    - 在下一次向 API 发起请求时，程序能够自动触发重量压缩流程。

- [ ] **手动压缩指令**
  - **测试输入**：在命令行中键入指令 `/compact`。
  - **预期观测**：
    - 终端控制台打印出压缩进行的系统提示。
    - 压缩成功后，输出提示展示压缩前后的 Token 估算大小变化。

- [ ] **三击失败熔断**
  - **测试输入**：模拟生成摘要时连续发生 3 次 LLM 请求超时或出错。
  - **预期观测**：
    - 控制台打印警告信息，并声明自动压缩已熔断禁用。
    - 对话能够继续交互，不再自动进入摘要生成流程。

---

## 4. 端到端功能验证

- [ ] **全流程端到端工作验证**
  - **测试输入**：
    1. 启动 `bingtangCode` 客户端。
    2. 执行返回超大文本的工具调用（如读取大文件），验证大工具结果自动落盘（轻量预防）。
    3. 持续对话使得对话累积大小超过自动压缩阈值，或者直接输入 `/compact` 强行压缩。
  - **预期观测**：
    - 对话历史被成功替换为一则结构化摘要 + 边界提示消息 + 近期保留消息。
    - 发送下一条用户消息，LLM 能够基于摘要和近期保留消息正确回答，并在需要查看旧文件时，主动调用 `read_file` 重新读取，没有根据摘要脑补代码。
