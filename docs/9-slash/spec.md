# 终端 AI 编程助手命令系统规格说明书 (spec.md)

## 背景
`bingtangCode` 作为一个终端 AI 编程助手，需要提供对一些与大模型无关的本地控制命令（如清屏、查看状态、切换模式）的支持。当前的命令处理采用散落在 TUI 和主循环中的 if-else 硬编码方式，缺乏扩展性，且容易因遗漏拦截而将非对话指令发给大模型，浪费 Token 且降低响应速度。

## 目标用户
使用 `bingtangCode` 助手的开发者，期望在终端中有顺畅、快捷、低延迟的辅助命令操作体验，且对 Token 消耗敏感。

## 能力清单
- **命令前缀拦截**：在输入入口自动拦截以斜杠开头的输入，绕过 AI 决策环。
- **集中声明式注册**：支持将每个命令的名称、别名、说明、类型及具体执行方法进行统一注册。
- **启动期冲突检测**：程序启动时扫描所有命令的名和别名，若存在重复则拒绝启动，尽早暴露开发错误。
- **无感参数解析**：解析器将命令与参数以首个空格为界进行拆分，且对命令名进行大小写无关的规范化处理。
- **执行模式分类**：命令分为纯本地命令（只读本地）、环境状态控制命令（改变 UI 或模式）以及提示词模版命令（向 AI 发送预设 Prompt）。
- **界面控制抽象解耦**：声明界面控制接口以解耦命令的逻辑和具体的控制台渲染细节。
- **防呆与智能引导**：当输入未知命令或缺少必填参数时，提供友好的系统引导信息，避免直接报错崩溃。
- **Tab 智能补全**：输入以斜杠开头的内容后按 Tab 键可触发匹配补全，唯一匹配直接上屏，多重匹配弹出交互式菜单供选择。
- **内置 12 个高频指令**：预置退出、模式切换（分为 plan 和 do 两条指令）、对话压缩、历史会话切换、会话详情显示、记忆状态展示、当前权限查询、重置清屏、代码审查请求以及命令帮助功能。

## 非功能要求
- **零网络交互延迟**：本地命令（本地及界面状态类）执行响应必须在毫秒级内完成，且完全不消耗大模型 API 的 Token。
- **防内存死锁**：输入解析在本地完成，对于空行或纯斜杠等畸形输入，必须实现提前拦截和防呆，杜绝发给大模型。
- **Tab 响应速度**：Tab 键触发自动匹配的时间应小于 50ms，且菜单切换流畅无顿挫。

## 设计骨架
- **数据模型**：命令元数据类、命令执行环境上下文、命令类型枚举、界面控制器接口。
- **服务组件**：命令注册中心（负责注册和防撞校验）、输入分流器（负责拦截与分支路由）、命令解析器。
- **UI 组件**：下拉选择交互框（ActionListBox，用于选择历史会话或多重补全候选）。

## Out of Scope (本期不做)
- 支持用户在运行期间动态新增自定义命令。
- 支持在运行时动态从网络下载或修改提示词模版。
- 细粒度的命令级用户访问权限控制。
