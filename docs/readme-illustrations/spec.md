# README 流程与结果图示说明 — 功能规格说明书（Spec）

## 背景
`bingtangCode` 是一个终端 AI 编程助手，随着 v0.3 至 v0.5 版本的迭代，项目已经具备了完整的 ReAct Agent Loop 运行时和五层防御权限系统。为了提升项目的易用性、可读性并让新用户能够快速直观地理解其核心架构与交互逻辑，需要在项目的根目录 `README.md` 中补充流程及结果的视觉化图示说明。

## 目标用户
- 首次访问项目的开源开发者，需要快速理解项目的工作原理。
- 打算使用或二次开发 `bingtangCode` 的用户，需要直观理解安全沙箱及权限拦截机制。

## 能力清单
- 提供一个 ReAct Agent Loop 的整体运行流程图，说明模型如何进行“思考-工具调用-结果观察”的循环。
- 提供一个五层防御权限系统（PermissionGate）的拦截流程图，说明五层拦截校验的先后顺序及生效范围。
- 提供一个终端交互概念效果图，直观展现 ASCII Banner、底部状态栏、模式切换和人在回路（HITL）交互选项。
- 将上述图片文件保存在指定目录中，并在根目录 of `README.md` 合理位置进行引用和文字说明排版。

## 非功能要求
- **风格一致性**：前两张图采用**纯净无人物的 3D 卡通微立体风格（糖果色系/卡通色调）**，与第三张图中的对话框、卡片以及整体质感高度一致。背景为温暖舒适的浅米白/暖乳白色。
- **高品质排版**：白色的圆角卡片带有细腻的软阴影、高光折射、彩色渐变边框，并包含精致的拟物化 3D 卡通图标。
- **图片自适应**：在 GitHub 的 Markdown 渲染下，图片尺寸应适中，在暗色模式与亮色模式下均可读。
- **文件路径规范**：所有图片必须存储在 `fig/readme/` 目录下，便于统一管理，不污染根目录。

## 设计骨架
- **ReAct Agent Loop（思考与执行循环）**：
  - 结构：**无人物 3D 卡通步骤流程图**。
  - 画面细节：在浅米色底色上，各阶段白色圆角方框排列成流程连线，包含 Loop 迭代判定（旋转箭头图标）与 Has Tool Call 判定，以及 Observation（监视器图标）回灌至 Loop Iteration 的循环反馈路径。

- **五层防御权限系统（PermissionGate）**：
  - 结构：**无人物 3D 卡通垂直卡片柜 + 侧边彩色实例标签**。
  - 画面细节：浅米色底色上垂直排列 5 个 3D 白色圆角卡片（代表 5 道防线，内部配有红色警报盾牌、保险箱、清单本、开关、小手指等 3D 拟物图标），左侧配有药丸形彩色实例标签。若拦截则由管道连向右侧红色 `Blocked / Access Denied` 按钮，通过则连向底部的绿色 `Tool Executed` 按钮。

- **终端交互界面设计（Terminal UI）**：
  - 结构：**场景化 3D 人物交互**（保持现状，作为人机互动的场景展示）。
  - 画面细节：前景为一名 3D 拟物角色坐在书桌前，面对着电脑屏幕敲击键盘。其屏幕上方或侧方弹出精致的 3D 磨砂玻璃质感对话框/气泡，展现 `BING TANG HULU` 启动字符画效果以及 HITL 人在回路确认弹窗交互按钮。
